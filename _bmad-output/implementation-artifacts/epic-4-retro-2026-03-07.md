## Challenges & Growth Areas

### DESIGN-SPEC Values Lost in Prose
The dev agent reads the DESIGN-SPEC but drifts on pixel-level details — outputting `h-1` (4px) instead of `height: 3px`, 10px dots instead of 5px. The spec has the values, but they're embedded in prose paragraphs. Nicolas identified this as a "memory issue" related to single source of truth — the source exists but isn't structured for reliable consumption.

**Resolution:** Story 5.0 will split DESIGN-SPEC into prose (rationale) + TOML (structured values). The TOML becomes the machine-readable contract.

### No Backlog for Deferred Items
The constellation SVG was deferred in Story 4-2 with a comment in the story file — not tracked anywhere. No dev backlog existed. Nicolas caught this: "deferred to when/where? Has it been added to the backlog? Do we even have a dev backlog?"

**Resolution:** Create `backlog.md` as part of Story 5.0. Sweep all Epic 1-4 story records for deferred items. Review backlog mandatory in Epic 5 retro.

### Hugo Gotchas Rediscovered
`if not` unreliable on slices (Story 4-4), `safeCSS` for dynamic CSS variables (Story 4-3). These keep appearing across stories.

**Resolution:** Document Hugo gotchas as part of Story 5.0 — add to project knowledge.

### Story 4-3 Review Scope Bleed
Story 4-3's code review expanded significantly — 15 issues including homepage wiring that was technically Story 4-4's scope. This pulled work forward but made 4-4 simpler.

**Nicolas's verdict (consistent with Epic 3):** Healthy pragmatism. Review catches scope issues. Net positive in this case.

## Key Insights & Lessons Learned

### 1. Split Specs into Prose + Structured Data
Design specs should have a structured companion (TOML) for measurable values. Prose for rationale, TOML for contract. Prevents LLM drift on fine details.

### 2. Track Deferred Items Formally
Comments in completed story files are not tracking. A lightweight backlog document prevents items from disappearing.

### 3. No Tech Debt Policy
Nicolas: "no design debt if we can avoid it." Sweep and document before moving forward. Story 5.0 absorbs all housekeeping.

### 4. Meta-Content Scales to System Documentation
Epic 4 documented the authoring workflow as a pattern. Epic 5 will document the entire ndb_hugo system as portfolio content — confronting the architecture while creating real value.

### 5. Reusable Skills as Portable Practices
Package learnings (design-spec splitting, content schemas) as skills that can be stored, shared, and reinstalled across projects. Survives BMAD reinstalls.
